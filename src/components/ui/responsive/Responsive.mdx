## Overview

Yorkie UI provides built-in responsive styles, eliminating the need for manually adding media queries and nested styles.
Instead, you can simply provide object and array values to add mobile-first responsive styles.

Yorkie UI defines default breakpoints using em unit based on the theme object, which are utilized by the responsive syntax.

```jsx
const breakpoints = {
  base: '0rem', // 0px
  sm: '40rem', // ~640px - base font size 16px
  md: '48rem', // ~768px
  lg: '64rem', // ~1024px
  xl: '80rem', // ~1280px
};
```

## The Array syntax

All style props accept arrays as values for mobile-first responsive styles. This is the recommended method.

To make the `width` or `w` responsive using the array syntax, here's what you need to do:

```jsx
<Box bg="orange.7" w={[300, 400, 500]} h={[100, 200, 300]}>
  Responsive a box
</Box>
```

## The Object syntax

You can define responsive values with breakpoint aliases in an object, where any undefined alias key will define the base, non-responsive value.

```jsx
<Text fontSize={{ base: '24px', md: '40px', lg: '56px' }}>Responsive a text</Text>
```

## Under the hook

Given the following, this shortcut serves as an alternative to manually writing media queries.

```jsx
<Box width={[1, 1 / 3, 1 / 7]} />
```
